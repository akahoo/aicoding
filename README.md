# 🃏 掼蛋 - 在线扑克游戏

一个基于 React + Node.js + Socket.IO 的网页版掼蛋游戏，支持 4 人在线对战。

## 🎮 游戏特色

- **4 人标准局**：经典掼蛋玩法，2v2 组队对战
- **无需登录**：系统自动分配昵称，支持自定义修改
- **邀请对战**：通过 URL 邀请好友加入同一牌局
- **实时对战**：WebSocket 实时通信，流畅游戏体验
- **完整牌型**：支持单张、对子、三张、顺子、连对、飞机、炸弹等所有牌型

## 🛠️ 技术栈

### 后端
- **Node.js** - 运行时
- **Express** - Web 框架
- **Socket.IO** - WebSocket 实时通信
- **内存存储** - 房间和牌局数据

### 前端
- **React 18** - UI 框架
- **Vite** - 构建工具
- **TailwindCSS** - 样式框架
- **Socket.IO Client** - 实时通信

## 🎯 游戏说明

### 基本规则
1. **人数**：4 人游戏，2v2 组队（对家为队友）
2. **牌数**：两副牌共 108 张，每人 27 张
3. **胜负**：一方两人先出完牌即获胜
4. **出牌**：顺时针出牌，可出牌或选择"不要"

### 支持牌型

| 牌型 | 说明 | 示例 |
|------|------|------|
| 单张 | 任意一张牌 | 3, A, 2, 小王 |
| 对子 | 两张相同牌 | 33, AA |
| 三张 | 三张相同牌 | 333, AAA |
| 三带二 | 三张 + 一对 | 33344 |
| 顺子 | 5 张连续牌（不含王） | 34567 |
| 连对 | 3 对以上连续对子 | 334455 |
| 飞机 | 2 个以上连续三张 | 333444 |
| 炸弹 | 四张相同牌 | 3333 |
| 王炸 | 大王 + 小王 | 大王小王（最大） |

### 牌型大小

- **王炸** > **炸弹** > **其他牌型**
- 同牌型比较权重：2 > A > K > Q > J > 10 > ... > 3
- 不同牌型无法比较（炸弹和王炸除外）

## 📱 使用流程

1. **创建房间**
   - 访问首页
   - 输入昵称（可选）
   - 点击"创建房间"

2. **邀请好友**
   - 复制房间链接
   - 发送给好友
   - 等待 4 人就位

3. **开始游戏**
   - 所有玩家点击"准备"
   - 房主点击"开始游戏"
   - 庄家先出牌

4. **出牌流程**
   - 点击选择要出的牌
   - 点击"出牌"按钮
   - 或点击"不要"跳过

## 🤝 多人游戏

- 房间号：8 位随机字符串
- 邀请方式：分享 URL 链接
- 最大人数：4 人
- 掉线处理：玩家可重新加入

## 📝 开发说明

### API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/health` | GET | 健康检查 |
| `/api/room/create` | POST | 创建房间 |
| `/api/room/:id/join` | POST | 加入房间 |
| `/api/room/:id` | GET | 获取房间状态 |
| `/api/nickname` | GET | 获取随机昵称 |

### Socket 事件

| 事件 | 方向 | 说明 |
|------|------|------|
| `join-room` | C→S | 加入房间 |
| `player-ready` | C→S | 玩家准备 |
| `start-game` | C→S | 开始游戏 |
| `play-hand` | C→S | 出牌 |
| `pass` | C→S | 不要 |
| `chat-message` | C→S | 聊天 |
| `game-state` | S→C | 游戏状态 |
| `hand-played` | S→C | 出牌广播 |
